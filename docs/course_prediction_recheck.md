# コース進入予測の再検討レポート

期間: 2025/01-12 を学習データ、2026/01-04 を評価データ。
評価指標: 艇単位の正解率(`予測コース == 実コース`)。

## 結論

**現状の `予測コース = 艇番` (枠番=コース固定) を維持する**。

時系列リーク無しで選手履歴を累積した上でルールを適用しても、2026 Q1+04 のいかなる構成でも艇単位正解率がベースラインを下回ったため、`prediction-preview.py` のロジック変更は推奨しない。

## ベースラインの実力

2026/01-04 の `予測コース = 艇番` の艇単位正解率は **0.9193 (91.93%)** で、前回のノートブック調査(2025年自己評価で 92.35%)とほぼ整合している。レース単位の全艇正解率は 84.63%。

| 区分 | n | 正解率 |
|---|---:|---:|
| 全体 | 68,919 | 0.9193 |
| 進入固定レースのみ | 1,930 | **0.9855** |
| 進入固定以外 | 66,989 | 0.9174 |

枠番別では 1枠 99.0% → 5枠 86.6% → 6枠 88.5% で、内枠ほど枠番=コースが成立しやすい。

レース場別では **江戸川が 99.3%** で別格。最低は蒲郡 87.7%、桐生 87.8%、宮島 88.2%。

ミスマッチの上位パターンは 6→5 (891件), 5→6 (842件), 4→5 (557件), 5→4 (472件) と、隣接枠の入替えが大半を占める。

## 検証した拡張ルール

`scripts/dev/course_history.py` で 2025+2026 を時系列でソートし、選手ごとに「自身のレース日時点までの累積前付け率/6コース率」を算出(自レース行は除外)。これを使って下記の 30+ 構成を 2026 Q1+04 で評価した。

ルール優先度: 進入固定 → 江戸川固定 → 6コース傾向選手 → 前付け傾向選手 → 残りは枠番昇順で空きコースを埋める(玉突き処理)。

代表的な構成と結果(`Δpt` はベースラインからの艇単位正解率の変化、improved/degraded はベースとの単行差し引き):

| 構成 | rule | Δpt | imp | deg | net |
|---|---:|---:|---:|---:|---:|
| 江戸川固定のみ | 0.9236 | +0.000 | 0 | 0 | 0 |
| 6コ ≥0.9 / n≥50 | 0.9226 | -0.098 | 49 | 115 | -66 |
| 6コ ≥0.8 / n≥50 | 0.9178 | -0.581 | 96 | 486 | -390 |
| 前付 ≥0.5 / n≥30 | 0.9226 | -0.092 | 27 | 89 | -62 |
| 前付 ≥0.35 / n≥30 | 0.9166 | -0.692 | 93 | 558 | -465 |
| Combo 6≥0.8 + 前≥0.5 | 0.9158 | -0.771 | 186 | 704 | -518 |

**全構成で Δpt が 0 または負**。最も保守的な前付け 0.5 (n>=30) でさえ 27 改善 / 89 悪化 で純減 -62。閾値を緩めるほど悪化幅が拡大する。

### なぜ高閾値でも当たらないか

履歴 6コース率 ≥0.9 (n≥50) で発火した 89 行(2026/01-04)を直接調べると:

- 実際に 6コース に入った割合: **28.1%** (履歴 90% に対して大幅劣化)
- 実際は 枠番=コース だった割合: 68.5%
- 内訳: 5枠での発火 75 件中、実 6コース率 29.3%; 4枠 13 件中 23.1%

履歴 前付け率 ≥0.5 (n≥30) で発火した 41 行も同様:

- 実際に内コース(実コース < 艇番)に入った割合: **39.0%**
- 実際は 枠番=コース だった割合: 58.5%

つまり選手の長期履歴で 90% / 50% を示しても、当該レースで同じ進入が再現される確率は 30% 前後しかなかった。進入は当該レースの 6艇の構成・他選手の傾向・スタート展示前の駆引きに左右されるため、**個人履歴単体では予測できない**。

加えて、6コース・前付けルールは1選手を動かすたびに玉突きで他の艇のコースも書き換わる。仮に候補選手で 30% 当たっても、外れた 70% のレースでは 2 艇(候補本人 + 玉突き先)を同時に外すので、悪化件数は改善件数の 2-3 倍で増加する。

### 江戸川固定ルールの効果はゼロ

江戸川は 2026 Q1+04 の 413 レースで枠番=コース正解率 99.3% (一致率)。ベースラインで既に枠番=コースを返しているので、明示ルールを足しても予測は変わらず Δpt = 0。可読性のためのコメント・テスト追加には意味があるが、精度向上にはならない。

### 進入固定ルールの効果もゼロ

進入固定 320 レース中、ベースラインの一致率は 98.55%。ルールを足しても出力は変わらず Δpt = 0。前付け/6コースルールが進入固定レースを誤って書き換えるのを防ぐガードレールとしては有用だが、本レポートのルール構成(進入固定を最優先)では既に保護されている。

## 推奨アクション

### 採用しない (主結論)

`prediction-preview.py` のコース予測ロジックは現状(`予測コース = 艇番`)を維持する。コードの変更は不要。

### 維持する場合のドキュメント追記 (推奨)

`scripts/prediction-preview.py` 内の `programs_long['予測コース'] = programs_long['艇番']` 行近辺に、根拠コメントを更新しておくと将来の再検討者の手戻りを防げる。

```python
# Course entry: always frame number = course (枠番=コース)
# 検討履歴: docs/notebooks/previews-v2.ipynb (2025自己評価) および
#   docs/course_prediction_recheck.md (2025学習→2026 Q1検証) で
#   選手履歴ベースの拡張ルール(前付け率/6コース率/江戸川固定)を試したが
#   いずれもベースライン 91.9% を下回った。玉突き処理の悪化が個人傾向の
#   情報量を上回るため、現状維持が最適。
programs_long['予測コース'] = programs_long['艇番']
```

### 次に有望そうな方向 (本レポートの範囲外)

精度向上を本気で追うなら、個人履歴ではなく **レース単位の構成情報** を入力に取った分類モデルが候補。具体的には:

1. レース 6艇の (登録番号, 級別, 年齢, 過去 6コース率/前付け率, 当該レース場での過去進入分布) をまとめて入力し、コース順列を予測する LambdaRank または Permutation Classifier。
2. 進入隊形(イン進入/カド/4対2 等)の事前確率をレース構成から推定し、各艇のコース割当を構成に応じて決める。
3. 当日のスタート展示が公開されてから(prediction-preview ではなく estimate.py 側で)補正する。これは preview 予測タスクではなく estimate タスクの拡張になる。

いずれも本リクエストの「ルール+選手履歴の拡張」の範囲を超えるため、別途タスク化を推奨。

## 付録: 分析スクリプト

- `scripts/dev/course_recheck.py` — programs と previews を結合して長形式に変換、ベースライン算出
- `scripts/dev/course_history.py` — 選手累積履歴を作りルール構成をスイープ
- 出力: `/tmp/course_recheck_results.csv` (全 30+ 構成の数値テーブル)

再現:
```bash
python3 scripts/dev/course_recheck.py    # 2025/2026 を /tmp/course_*.pkl にキャッシュ
python3 scripts/dev/course_history.py    # 累積履歴 + ルールスイープ
```
